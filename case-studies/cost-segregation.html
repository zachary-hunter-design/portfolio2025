Designing a Specialized Mapping Tool to Streamline Property Analysis Workflows

Eliminating manual data entry and reducing site visit time through targeted UX design.

Executive Summary

I designed a specialized mapping feature for Cost Segregation Authority's property analysis app, transforming how analysts measure and catalog exterior property assets during site visits. The solution replaced cumbersome external tools and manual spreadsheet entry with an integrated, purpose-built mapping interface.

The new mapping tool delivered significant workflow improvements: streamlined property measurement processes, eliminated double data entry, reduced time spent switching between tools, and enhanced accuracy in property cataloging.

The project successfully transformed a manual, error-prone process into an efficient, integrated workflow that supports analysts' specialized requirements for tax-advantaged property assessments.

Introduction

What does cost segregation even mean?

Cost segregation is like breaking down a big building's cost into smaller pieces to see if some parts can help you save money on taxes. Cost Segregation Authority's analysts use this app to catalog and break down property assets to see how they can best assist their clients.

Think of it like this: Imagine you order a big pizza with lots of different toppings. Instead of paying for the entire pizza in one go, you can separate the cost of each topping. That way, you can decide how much you want to pay for the cheese, the pepperoni, the mushrooms, and so on.

Designing My Solution

How regular feedback secured success

I set up weekly design reviews with the Cost Segregation Authority management team and a handful of analysts—think of it as my regular dose of "here's what's working and what's not." This tight feedback loop kept everything humming along nicely. When tweaks were needed (and they always are), I could usually knock them out within 24 hours of our chat. Turns out frequent check-ins and solid feedback make for surprisingly speedy turnarounds.

Creating the Color Code

Since legal codes rarely get updates (shocking, right?), the property items we can measure are pretty much set in stone—and we can only count what's visible from the outside using our mapping tool. So I figured, why not make life easier with some color-coding?

Plot twist: Cost Segregation Authority already had their own color system, so I grabbed that too. I sorted everything into 3 categories and whipped up some handy spreadsheets breaking down each property item with its color and measurement icon. Now the developers have a nice cheat sheet to reference during implementation instead of drowning in legal jargon.

The Palette Side Menu

(note: Cost Segregation Authority calls property items "takeoffs")

Users wrangle all their map additions from this trusty side menu, designed to feel familiar since it mirrors the external tool they already know and love. Adding property items? Easy—just search or hit the "add takeoff" button to summon the overlay. iPad users get a neat swipe-left trick on menu items for quick delete/edit options.

Select a takeoff and watch its map buddies light up like a Christmas tree. Need more screen real estate? Tap the chevron to minimize the menu—don't worry, your selections stick around and the toolbar stays put. The menu gracefully expands as you add takeoffs until it maxes out the screen space, then becomes scrollable because nobody likes cramped lists.
