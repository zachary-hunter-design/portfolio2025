Creating a Flexible Design System "Template"

Accelerating client project delivery through reusable design infrastructure.

Executive Summary

I developed a foundational design system template in Figma for a custom development firm, establishing a reusable framework that accelerates client project delivery while maintaining design consistency. The solution transformed the team's approach from building design systems from scratch to rapidly customizing pre-built components, tokens, and primitives tailored to each client's brand requirements.

The new template system delivered substantial efficiency gains: reduced initial design phase timelines, eliminated redundant component creation across projects, minimized inconsistencies between client deliverables, and enabled designers to focus on customization rather than foundational structure building.

The project successfully established a scalable design workflow that balances speed with customization, providing the development firm with a competitive advantage in client onboarding while ensuring each project maintains its unique brand identity and functional requirements.

Primitives

Color System

All of the base primitive color values are divided into 4 main sections— Grays, Primary, Secondary, and Functional. Each has a color name and value attached to a hexcode, the lower the number, the lighter the color.

Numerical Primitives

The spacing values are incremented by whole factors of 8, with two exceptions 4 and 12 which I find valuable for spacing on smaller devices.

Radius values are also selected from factors of 8 but are assigned "tee-shirt sizes" names for ease of use.

Design Tokens

How to create a system that is flexible and consistent?

I established the design system foundation by first defining core primitives—basic values like spacing increments, border radius measurements, and color hex codes—then systematically organized these into semantic design tokens with function-based naming conventions. This approach allowed me to create tokens like "Button-Primary" and "Spacing-MD" that clearly communicate their intended use rather than their underlying values, making it intuitive for designers to select appropriate elements while maintaining consistency.

By separating the primitive values from their semantic applications, the system becomes highly scalable and rebrandable, as teams can simply update the core primitives to transform the entire design system's appearance while preserving the logical structure and component relationships.

Typography System

Text styles established with intutive names

I developed a comprehensive typography system by creating text styles with clear, semantic names that immediately communicate their intended purpose and context within the interface. This approach produced an intuitive library where styles like "H1 Header Bold," "Button Text Regular Sm," and "Label Text Lg Regular" tell designers exactly when and where to use each variant, eliminating guesswork and ensuring consistent typography across all touchpoints.

The descriptive naming convention allows teams to quickly navigate the text hierarchy while maintaining design coherence, and makes it simple to customize font properties for different clients without losing the logical structure that guides proper typographic application throughout their projects.

Layout Grids

I created standardized layout grinds for all platforms

There is honestly nothing special about these, they conform to common and best practices.

Creating components

I created prototype ready base components for the system

I assembled the foundational design tokens and text styles into comprehensive component sets that showcase all variations and states in one organized system. These component libraries include buttons across multiple sizes and states, form inputs with different configurations, toggles, cards, and avatar systems that demonstrate how the atomic elements work together cohesively.

By creating these complete component sets, designers can quickly customize entire interface patterns by adjusting the underlying tokens, ensuring consistent implementation while dramatically reducing the time needed to build custom interfaces for each client project.
